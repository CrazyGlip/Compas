
# ПРАВИЛА РАЗРАБОТКИ (DEVELOPMENT RULES)

⚠️ **КРИТИЧЕСКИ ВАЖНОЕ ПРАВИЛО / CRITICAL RULE** ⚠️

Любые изменения, касающиеся следующих аспектов, **СТРОГО ЗАПРЕЩЕНЫ** без предварительного явного согласования с пользователем:

1.  **Изменение интерфейса (UI)**:
    *   Модификация глобальных лейаутов.
    *   Изменение навигации.
    *   Существенные изменения визуального стиля.

2.  **Структура приложения**:
    *   Создание новых экранов (Screens).
    *   Удаление существующих экранов.
    *   Создание или удаление крупных компонентов.

3.  **Файловая система**:
    *   Удаление файлов.
    *   Полная перезапись файлов (кроме случаев, когда это необходимо для внедрения согласованной фичи).

4.  **База данных (Data Integrity)**:
    *   **ЗАПРЕЩЕНО** изменять или перезаписывать столбец `image_url` в таблице `specialties`. Этот столбец управляется пользователем вручную. Все программные обновления (Seed/Sync/Update) должны игнорировать это поле.

**ПРОТОКОЛ ДЕЙСТВИЙ:**

Если запрос пользователя подразумевает вышеуказанные действия:
1.  **ОСТАНОВИТЬСЯ**. Не генерировать код сразу.
2.  **ОПИСАТЬ** план изменений (Спецификацию) на естественном языке.
3.  **СПРОСИТЬ** разрешение пользователя на выполнение этого плана.
4.  Только после получения явного "Да" или "Согласен" генерировать блок `<changes>`.

*Исключение:* Исправление багов, оптимизация логики внутри функций или мелкие правки контента, которые не ломают текущую структуру UI.
